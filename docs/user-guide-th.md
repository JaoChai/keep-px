## 1. เริ่มต้นใช้งาน

### 1.1 สมัครและเข้าสู่ระบบ

- เข้าเว็บ Keep-PX แล้วกด **"เข้าสู่ระบบด้วย Google"**
- ระบบจะสร้างบัญชีให้อัตโนมัติจาก Google Account ของคุณ
- เมื่อเข้าสู่ระบบสำเร็จ จะถูกพาไปที่หน้า **แดชบอร์ด** ทันที

### 1.2 หลังเข้าสู่ระบบครั้งแรก ควรทำอะไรก่อน?

1. ไปที่ **Pixels** → สร้าง Pixel ตัวแรก
2. ไปที่ **เซลเพจ** → สร้างเซลเพจแรก เชื่อม Pixel ที่สร้างไว้
3. เผยแพร่เซลเพจ → แชร์ลิงก์ให้ลูกค้า
4. กลับมาดูข้อมูลที่หน้า **Events** และ **แดชบอร์ด**

## 2. แดชบอร์ด

หน้าแดชบอร์ดแสดงภาพรวมการทำงานทั้งหมด ประกอบด้วย:

### 2.1 ตัวเลขสรุป (5 การ์ด)

| การ์ด | ความหมาย |
|-------|----------|
| **Active Pixels** | จำนวน Pixel ที่เปิดใช้งานอยู่ / จำนวนทั้งหมด |
| **Events Today** | จำนวน Event ที่เข้ามาวันนี้ (แสดงแนวโน้มเทียบวันก่อน) |
| **CAPI Rate** | อัตราการส่ง Event ไปยัง Facebook สำเร็จ (เขียว = ดี, เหลือง = ปานกลาง, แดง = มีปัญหา) |
| **Events This Week** | จำนวน Event สัปดาห์นี้ |
| **Active Replays** | จำนวน Replay ที่กำลังทำงานอยู่ |

### 2.2 โควตา Event รายเดือน

- แถบแสดงจำนวน Event ที่ใช้ไป เทียบกับ Limit ของแพ็กเกจ
- ถ้าใกล้เต็มให้พิจารณาอัปเกรดแพ็กเกจ

### 2.3 กราฟ Event Volume

- กราฟแสดงแนวโน้ม Event ตามเวลา
- เลือกดูได้: 7 วัน, 14 วัน, 30 วัน, 90 วัน

### 2.4 กิจกรรมล่าสุด

- แสดง 8 Event ล่าสุดที่เข้ามา
- แต่ละ Event แสดง: ชื่อ Event, Pixel ที่รับ, สถานะ CAPI, เวลา

### 2.5 สถานะ Pixel

- รายการ Pixel ทั้งหมดพร้อมสถานะ Active/Paused

### 2.6 Event ยอดนิยม

- แท่งกราฟแสดง Event ที่เกิดบ่อยที่สุด (PageView, ViewContent, Purchase ฯลฯ)

## 3. จัดการ Pixel

> เมนู **Pixels** ในแถบด้านซ้าย

### 3.1 สร้าง Pixel ใหม่

กดปุ่ม **"สร้าง Pixel"** แล้วกรอกข้อมูล:

| ช่อง | คำอธิบาย | ตัวอย่าง |
|------|----------|----------|
| **ชื่อ Pixel** | ชื่อที่ตั้งเอง ไว้จำง่าย ๆ | "Pixel โฆษณาเสื้อผ้า" |
| **Facebook Pixel ID** | รหัส Pixel จาก Facebook Events Manager | 123456789012345 |
| **Access Token** | โทเค็นสำหรับส่งข้อมูลไป Facebook CAPI | EAAxxxxxxxx... |
| **Test Event Code** | (ไม่บังคับ) โค้ดทดสอบจาก Events Manager | TEST12345 |
| **Backup Pixel** | (ไม่บังคับ) Pixel สำรองที่จะรับ Event ซ้ำอัตโนมัติ | เลือกจาก Pixel ที่มีอยู่ |

### 3.2 หา Pixel ID และ Access Token ได้ที่ไหน?

1. เข้า **Facebook Events Manager** → เลือก Pixel ของคุณ
2. **Pixel ID**: แสดงอยู่ใต้ชื่อ Pixel (เลข 15-16 หลัก)
3. **Access Token**: ไปที่ Settings → Generate Access Token

### 3.3 ตาราง Pixel

| คอลัมน์ | ความหมาย |
|---------|----------|
| **ชื่อ** | ชื่อที่คุณตั้งไว้ |
| **Pixel ID** | รหัส Facebook Pixel (แสดงบางส่วน) |
| **สถานะ** | Active (เปิด) / Paused (หยุดชั่วคราว) — กดสลับได้ |
| **Backup Pixel** | Pixel สำรอง (ถ้าตั้งไว้) |
| **วันที่สร้าง** | วันที่สร้าง Pixel ในระบบ |

### 3.4 การดำเนินการกับ Pixel

| ปุ่ม | ทำอะไร |
|------|--------|
| **Test Connection** | ทดสอบการเชื่อมต่อกับ Facebook CAPI ว่าใช้งานได้ |
| **แก้ไข** | แก้ไขข้อมูล Pixel |
| **ลบ** | ลบ Pixel (Event ที่เก็บไว้แล้วจะยังอยู่) |
| **สลับสถานะ** | เปิด/ปิด Pixel ชั่วคราว (ไม่ต้องลบ) |

### 3.5 Backup Pixel คืออะไร?

- เมื่อเปิด Backup Pixel ระบบจะส่ง Event ไปยัง Pixel สำรอง **พร้อมกัน** กับ Pixel หลัก
- ประโยชน์: ถ้า Pixel หลักโดนแบน ยังมีข้อมูลอยู่ใน Pixel สำรอง
- วิธีตั้ง: สร้าง Pixel ที่ 2 ก่อน → แก้ไข Pixel ที่ 1 → เลือก Pixel ที่ 2 เป็น Backup

## 4. ดู Events (เหตุการณ์)

> เมนู **Events** ในแถบด้านซ้าย

หน้า Events มี **2 โหมด**:

### 4.1 โหมด Live (เรียลไทม์)

ดู Event ที่เข้ามาแบบสด ๆ ขณะที่ลูกค้าเข้าเซลเพจ

**ปุ่มควบคุม:**
- ▶️ Play / ⏸ Pause — เริ่ม/หยุดรับ Event สด
- 🔄 Refresh — โหลดข้อมูลใหม่
- 🗑 Clear — ล้างรายการ Event ในหน้าจอ

**ข้อมูลที่แสดง:**
- จำนวน Event ในเซสชัน
- อัตรา Event ต่อนาที
- CAPI Forwarding Rate
- กราฟ Event Rate แบบ 5 นาที
- สัดส่วน Event แยกตามประเภท

**ตาราง Event:**

| คอลัมน์ | ตัวอย่าง |
|---------|---------|
| **Event Name** | PageView, Purchase, Lead, ViewContent |
| **Pixel** | "Pixel โฆษณาเสื้อผ้า" |
| **Source URL** | หน้าเว็บที่ Event เกิด |
| **CAPI** | ✅ ส่งสำเร็จ / ❌ ส่งไม่สำเร็จ |
| **เวลา** | 2 นาทีที่แล้ว |

### 4.2 โหมด History (ประวัติ)

ดู Event ย้อนหลังทั้งหมด

- **Filter ตาม Pixel** — เลือกดูเฉพาะ Pixel ที่ต้องการ
- **Pagination** — แสดง 50 Event ต่อหน้า เลื่อนหน้าได้
- ข้อมูลเหมือนโหมด Live แต่แสดงวันเวลาเต็ม

## 5. Replay Center (ส่ง Event ซ้ำ)

> เมนู **Replay** ในแถบด้านซ้าย

### 5.1 Replay คืออะไร?

Replay คือการนำ Event ที่เก็บไว้ในระบบ ส่งซ้ำไปยัง Pixel ตัวใหม่ผ่าน Facebook CAPI

**ใช้เมื่อไหร่?**
- แอดเคาท์โดนแบน ต้องย้ายข้อมูลไปยัง Pixel ใหม่
- อยากส่ง Event ซ้ำเพื่อให้ Facebook เรียนรู้ข้อมูลได้เร็วขึ้น

### 5.2 วิธีสร้าง Replay

**ขั้นที่ 1: กรอกข้อมูล (แผงซ้าย)**

| ช่อง | คำอธิบาย |
|------|----------|
| **Source Pixel** | Pixel ต้นทาง (จะดึง Event จาก Pixel นี้) |
| **Target Pixel** | Pixel ปลายทาง (จะส่ง Event ไปยัง Pixel นี้) |
| **Event Type** | (ไม่บังคับ) เลือก Replay เฉพาะ Event บางประเภท เช่น Purchase เท่านั้น |
| **Date Range** | (ไม่บังคับ) กำหนดช่วงเวลาของ Event ที่ต้องการ |
| **Time Mode** | "Original" = ใช้เวลาเดิมของ Event / "Current" = ใช้เวลาปัจจุบัน |
| **Batch Delay** | หน่วงเวลาระหว่างการส่งแต่ละชุด (0-60,000 ms) |

**เลือก Time Mode:**
- **Original** — เหมาะกับ Event ที่เกิดภายใน 7 วัน Facebook ยอมรับเวลาเดิม
- **Current** — เหมาะกับ Event เก่ากว่า 7 วัน เพราะ Facebook อาจปฏิเสธ Event ที่เก่าเกินไป

**ขั้นที่ 2: Preview (ดูตัวอย่าง)**

กดปุ่ม **"Preview"** ระบบจะแสดง:
- จำนวน Event ที่จะถูก Replay
- คำเตือนถ้า Event เก่าเกินไป
- ตัวอย่าง Event ที่จะส่ง
- กดยืนยันเพื่อเริ่ม Replay

**ขั้นที่ 3: ติดตามสถานะ (แผงขวา)**

| สถานะ | ความหมาย |
|-------|----------|
| **Pending** | รอเริ่ม |
| **Running** | กำลังส่ง Event อยู่ |
| **Completed** | ส่งเสร็จเรียบร้อย |
| **Failed** | ส่งไม่สำเร็จ (ดูข้อผิดพลาด) |
| **Cancelled** | ถูกยกเลิก |

**ปุ่มที่ใช้ได้:**
- **Cancel** — ยกเลิก Replay ที่กำลังทำงาน
- **Retry Failed** — ส่ง Event ที่ล้มเหลวซ้ำอีกครั้ง

### 5.3 Replay Credit

- การ Replay แต่ละครั้งใช้ **1 Credit**
- ดูจำนวน Credit คงเหลือได้ที่หน้า Replay หรือหน้า Billing
- ซื้อเพิ่มได้ที่หน้า **Billing → แท็บ Replays**

## 6. เซลเพจ (Sale Pages)

> เมนู **เซลเพจ** ในแถบด้านซ้าย

### 6.1 เซลเพจคืออะไร?

เซลเพจคือหน้าเว็บที่ Keep-PX สร้างและโฮสต์ให้ ใช้สำหรับ:
- แสดงสินค้า/บริการ
- **เก็บข้อมูล Pixel อัตโนมัติ** เมื่อมีคนเข้าชม
- แชร์ลิงก์ไปยัง Social Media หรืออีเมล

**ลิงก์เซลเพจ:** `https://yourdomain.com/p/ชื่อ-url-ที่ตั้ง`

### 6.2 สร้างเซลเพจ

กดปุ่ม **"สร้างเซลเพจ"** เลือก Template:

| Template | คำอธิบาย |
|----------|----------|
| **Classic** | แบบตายตัว กรอกข้อมูลตามช่อง เรียบง่าย |
| **Blocks** | แบบลาก-วาง ปรับแต่งอิสระ ยืดหยุ่นกว่า |

### 6.3 ตั้งค่าเซลเพจ (Classic Template)

#### ข้อมูลพื้นฐาน

| ช่อง | คำอธิบาย |
|------|----------|
| **ชื่อหน้า** | ชื่อภายใน ไว้จำ (ลูกค้าไม่เห็น) |
| **URL Slug** | ส่วนท้ายของลิงก์ เช่น `my-product` → `/p/my-product` (ถ้าไม่กรอก ระบบสร้างให้อัตโนมัติ) |
| **เชื่อม Pixel** | เลือก Pixel ที่ต้องการเก็บข้อมูล (เลือกได้หลายตัว) |

#### ส่วน Hero (หัวหน้า)

| ช่อง | คำอธิบาย |
|------|----------|
| **หัวข้อ** | ชื่อสินค้า/บริการ |
| **คำบรรยาย** | คำอธิบายสั้น ๆ |
| **รูปภาพ Hero** | อัปโหลดรูปหรือใส่ URL |

#### ส่วนรายละเอียด

| ช่อง | คำอธิบาย |
|------|----------|
| **คำอธิบาย** | เนื้อหาสินค้า |
| **จุดเด่น** | รายการ Feature (ใส่ได้สูงสุด 10 ข้อ) |
| **รูปภาพเพิ่มเติม** | อัปโหลดรูปแบบ Grid |

#### ปุ่ม Call-to-Action (CTA)

| ช่อง | คำอธิบาย |
|------|----------|
| **ข้อความบนปุ่ม** | เช่น "ซื้อเลย", "สมัครเลย" |
| **ลิงก์ปุ่ม** | URL ที่ลูกค้าจะถูกพาไปเมื่อกด |

#### การติดตาม (Tracking Settings)

| ช่อง | คำอธิบาย |
|------|----------|
| **CTA Event** | Event ที่จะยิงเมื่อกดปุ่ม CTA: Lead / Purchase / Contact / CompleteRegistration |
| **Content Name** | ชื่อสินค้า (ส่งให้ Facebook) |
| **Content Value** | ราคาสินค้า (ส่งให้ Facebook) |
| **Currency** | สกุลเงิน: THB, USD |

**Event ที่ยิงอัตโนมัติ:**
- **PageView** — เมื่อลูกค้าเปิดหน้าเซลเพจ
- **ViewContent** — เมื่อลูกค้าเลื่อนดูเนื้อหา
- **Contact** — เมื่อลูกค้ากดลิงก์ LINE หรือโทรศัพท์

#### การตกแต่ง

| ช่อง | คำอธิบาย |
|------|----------|
| **สีพื้นหลัง** | สีหลักของหน้า |
| **สีเน้น** | สีปุ่มและองค์ประกอบเด่น |
| **สีตัวอักษร** | สีข้อความ |
| **รูปพื้นหลัง** | URL รูปพื้นหลัง (ถ้ามี) |

#### ข้อมูลติดต่อ

| ช่อง | คำอธิบาย |
|------|----------|
| **LINE ID** | ไอดีไลน์สำหรับให้ลูกค้าติดต่อ |
| **เบอร์โทร** | เบอร์โทรศัพท์ |
| **เว็บไซต์** | URL เว็บไซต์หลัก |

### 6.4 เผยแพร่เซลเพจ

| ปุ่ม | ความหมาย |
|------|----------|
| **Save as Draft** | บันทึกไว้ก่อน ยังไม่เผยแพร่ |
| **Publish** | เผยแพร่ทันที — ลูกค้าเข้าถึงได้ผ่านลิงก์ |

### 6.5 การทำงานของเซลเพจเมื่อลูกค้าเข้าชม

```
ลูกค้าเปิดลิงก์ /p/my-product
    ↓
ระบบยิง PageView + ViewContent → Facebook CAPI
    ↓
ลูกค้าเลื่อนดูสินค้า อ่านรายละเอียด
    ↓
ลูกค้ากดปุ่ม "ซื้อเลย"
    ↓
ระบบยิง Purchase (หรือ Lead/Contact) → Facebook CAPI
    ↓
ลูกค้าถูกพาไปยังลิงก์ที่ตั้งไว้
```

### 6.6 ตารางเซลเพจ

| คอลัมน์ | ความหมาย |
|---------|----------|
| **ชื่อ** | ชื่อเซลเพจ |
| **URL** | ลิงก์เซลเพจ |
| **Pixels** | จำนวน Pixel ที่เชื่อมอยู่ |
| **Template** | Blocks หรือ Classic |
| **สถานะ** | Draft / Published |

**ปุ่ม:** Preview (ดูตัวอย่าง) / Edit (แก้ไข) / Delete (ลบ)

## 7. Billing (แพ็กเกจและการชำระเงิน)

> เมนู **Billing** ในแถบด้านซ้าย

### 7.1 ภาพรวมบัญชี

การ์ดหลักแสดง:
- **แพ็กเกจปัจจุบัน** ของคุณ
- **สถานะบัญชี** (Active / Suspended)
- **Event ที่ใช้ไป** เทียบกับโควตา
- **Replay Credit คงเหลือ**

### 7.2 แพ็กเกจ (แท็บ Plans)

| แพ็กเกจ | Pixel | เซลเพจ | Event/เดือน | Replay Credit | ราคา |
|---------|-------|---------|-------------|---------------|------|
| **Sandbox** (ฟรี) | จำกัด | จำกัด | จำกัด | - | ฟรี |
| **Launch** | มากขึ้น | มากขึ้น | 1M | รวมมาให้ | มีค่าใช้จ่าย |
| **Shield** | มากขึ้น | มากขึ้น | 5M | รวมมาให้ | มีค่าใช้จ่าย |
| **Vault** | ไม่จำกัด | ไม่จำกัด | ไม่จำกัด | รวมมาให้ | มีค่าใช้จ่าย |

กดปุ่ม **"Upgrade"** เพื่ออัปเกรดแพ็กเกจ

### 7.3 ซื้อ Replay Credit (แท็บ Replays)

| แพ็ก | จำนวน Replay |
|------|-------------|
| **แพ็ก 1** | 1 ครั้ง |
| **แพ็ก 3** | 3 ครั้ง |
| **Unlimited** | ไม่จำกัด |

## 9. คำศัพท์สำคัญ

### Facebook Pixel

โค้ดติดตามจาก Facebook ที่ใช้เก็บข้อมูลพฤติกรรมลูกค้าบนเว็บไซต์ ประกอบด้วย:
- **Pixel ID** — รหัส Pixel (ตัวเลข 15-16 หลัก)
- **Access Token** — กุญแจสำหรับส่งข้อมูลไป Facebook

### Event (เหตุการณ์)

การกระทำที่ลูกค้าทำบนเว็บไซต์ เช่น:

| Event | ความหมาย |
|-------|----------|
| **PageView** | เปิดหน้าเว็บ |
| **ViewContent** | ดูเนื้อหาสินค้า |
| **Lead** | สนใจสินค้า (กรอกฟอร์ม, กดดูรายละเอียด) |
| **Purchase** | ซื้อสินค้า |
| **Contact** | ติดต่อร้าน (กด LINE, กดโทร) |
| **CompleteRegistration** | สมัครสมาชิกสำเร็จ |

### Conversions API (CAPI)

วิธีส่งข้อมูล Event ไป Facebook ผ่านเซิร์ฟเวอร์ (Server-to-Server) ซึ่ง:
- เสถียรกว่า Pixel บนเว็บ (ไม่โดน Ad Blocker บล็อก)
- ข้อมูลแม่นยำกว่า
- Keep-PX ใช้ CAPI เป็นวิธีหลักในการส่ง Event

### Backup Pixel (Pixel สำรอง)

Pixel ตัวที่ 2 ที่รับ Event พร้อม ๆ กับ Pixel หลัก เป็นการสำรองข้อมูลไว้ในกรณีที่ Pixel หลักโดนแบน

### Replay (การส่ง Event ซ้ำ)

นำ Event ที่เก็บในระบบส่งซ้ำไปยัง Pixel ตัวใหม่ ใช้เมื่อ:
- แอดเคาท์โดนแบน ต้องสร้าง Pixel ใหม่
- อยากให้ Pixel ใหม่มีข้อมูลการเรียนรู้ (Learning Phase) เร็วขึ้น

### Replay Credit

หน่วยนับจำนวนครั้งที่สามารถ Replay ได้ ซื้อเพิ่มได้ที่หน้า Billing

### Event Quota (โควตา Event)

จำนวน Event สูงสุดที่รับได้ต่อเดือน ขึ้นอยู่กับแพ็กเกจ ถ้าเกินโควตาจะไม่สามารถรับ Event ใหม่ได้

## 10. ขั้นตอนการใช้งานแบบเต็ม

### สถานการณ์ที่ 1: เริ่มต้นใช้งานครั้งแรก

```
1. เข้าสู่ระบบด้วย Google
         ↓
2. ไปที่ Pixels → สร้าง Pixel (กรอก Pixel ID + Access Token)
         ↓
3. กด Test Connection ยืนยันว่าเชื่อมต่อ Facebook ได้
         ↓
4. ไปที่เซลเพจ → สร้างเซลเพจ
         ↓
5. เลือก Template → กรอกข้อมูลสินค้า → เชื่อม Pixel
         ↓
6. กด Publish → คัดลอกลิงก์ /p/xxx
         ↓
7. แชร์ลิงก์ไปยัง Facebook, LINE, อีเมล
         ↓
8. กลับมาดู Events → เปิดโหมด Live → ดู Event เข้ามาแบบสด
         ↓
9. ดูภาพรวมที่ Dashboard
```

### สถานการณ์ที่ 2: แอดเคาท์โดนแบน — กู้คืนข้อมูล

```
1. สร้างแอดเคาท์ Facebook ใหม่ + สร้าง Pixel ใหม่
         ↓
2. ไปที่ Pixels → สร้าง Pixel ใหม่ในระบบ (กรอก Pixel ID + Access Token ใหม่)
         ↓
3. ไปที่ Replay Center
         ↓
4. เลือก Source = Pixel เก่า, Target = Pixel ใหม่
         ↓
5. (ถ้าต้องการ) กรอง Event Type เช่น เอาเฉพาะ Purchase
         ↓
6. (ถ้า Event เก่ากว่า 7 วัน) เลือก Time Mode = "Current"
         ↓
7. กด Preview → ตรวจดูจำนวน Event → กดยืนยัน
         ↓
8. รอ Replay ทำงาน → ดูสถานะที่แผงขวา
         ↓
9. เสร็จแล้ว! Pixel ใหม่มีข้อมูล Event แล้ว
```

### สถานการณ์ที่ 3: ป้องกันล่วงหน้าด้วย Backup Pixel

```
1. สร้าง Pixel หลัก (Pixel A)
         ↓
2. สร้าง Pixel สำรอง (Pixel B) ในแอดเคาท์อื่น
         ↓
3. แก้ไข Pixel A → เลือก Pixel B เป็น Backup Pixel
         ↓
4. ตอนนี้ทุก Event จะถูกส่งไปทั้ง Pixel A และ Pixel B พร้อมกัน
         ↓
5. ถ้า Pixel A โดนแบน → Pixel B ยังมีข้อมูลครบ
         ↓
6. แก้ไขเซลเพจ → เปลี่ยนมาใช้ Pixel B เป็นหลัก
```
